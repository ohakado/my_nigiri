────────────────────────────────────────
  1. KomoringHeights とは
────────────────────────────────────────

  - 作者   : komori-n 氏
  - 言語   : C++
  - ライセンス: オープンソース（GitHub公開）
  - ベース : やねうら王（指し手生成等のデータ構造を流用）
  - アルゴリズム: df-pn（depth-first proof-number search）
                  + 証明駒／反証駒による探索ノード数削減
                  + n手詰めルーチン

  - 性能   : ミクロコスモス（1525手詰）を約10分で解答可能
  - 実装規模: 新規コード約1500行
  - GitHub : https://github.com/komori-n/KomoringHeights


────────────────────────────────────────
  2. 組み込み可否の結論
────────────────────────────────────────

  結論: ブラウザゲームへの直接組み込みは【困難・非推奨】

  理由:
  ① C++ バイナリのため、ブラウザ上でそのままは動作しない
  ② WebAssembly（WASM）へのコンパイルは理論上可能だが、
     「やねうら王」ごとビルドが必要で工数が膨大
  ③ 動作に 1GB 以上のメモリが必要なため、
     ブラウザ環境では現実的でない
  ④ KomoringHeights の真価は 100手・1000手級の超難問向けであり、
     握り詰め（3〜7手）には明らかにオーバースペック

────────────────────────────────────────
  4. 今後の改善方針（推奨）
────────────────────────────────────────

  ■ 短期（現状維持・微改善）
    - 現在の Alpha-Beta ソルバーで 3〜7手詰めは十分対応可能
    - タイムアウト上限（現在 300,000 ノード）のチューニング

  ■ 中期（性能強化が必要になった場合）
    - df-pn アルゴリズムを JavaScript で自前実装
    - 置換表（Transposition Table）の追加だけでも
      Alpha-Beta より大幅に高速化が見込める
    - Web Worker への移行で UI フリーズも解消可能

  ■ 長期（サーバーサイド化が許容できる場合）
    - KomoringHeights をサーバーで動かし、
      REST API 経由でブラウザから呼び出す構成も選択肢
    - ただし、握り詰め用途では費用対効果が低い


────────────────────────────────────────
  5. 作業再開時のチェックリスト
────────────────────────────────────────

  [ ] 現状の Alpha-Beta ソルバーで 7手詰めの動作を確認
  [ ] UI フリーズが問題になる場合 → Web Worker 化を検討
  [ ] さらなる高速化が必要な場合 → df-pn の JS 実装を検討
  [ ] KomoringHeights の WASM 化は工数大のため保留
